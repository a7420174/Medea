# ============================================================================
# OPTIONAL: vLLM / OpenAI-compatible Backend
# ============================================================================
# vLLM serves an OpenAI-compatible API — no code changes needed.
#
# Start vLLM:
#   vllm serve Qwen/Qwen3.5-14B-AWQ --max-model-len 16384 --gpu-memory-utilization 0.92 --enforce-eager
#
# Then set:
#   LLM_PROVIDER_NAME=OpenAI
#   OPENAI_API_BASE=http://localhost:8000/v1
#   OPENAI_API_KEY=dummy
#   BACKBONE_LLM=Qwen/Qwen3.5-14B-AWQ
#
# GPU memory estimates (AWQ 4-bit):
#   Qwen3.5-7B-AWQ  + 16K ctx → ~8GB  (T4 safe)
#   Qwen3.5-14B-AWQ + 16K ctx → ~14GB (T4 tight, recommended)
#   Qwen3.5-32B-AWQ + 16K ctx → ~24GB (A100 40GB)
#
# Also works with Cerebras and Groq via built-in provider support.
